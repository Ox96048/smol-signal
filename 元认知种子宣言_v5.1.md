## 一、一个同构问题

为什么人类四千年制度积累后，三百年内爆发了工业革命？
为什么神经网络在漫长梯度下降后，损失会突然骤降？
为什么一个 AI 能随口掏出 `{D969A300-E7FF-11d0-A93B-00A0C90F2719}` 这种冷门注册表咒语，
却说不清自己什么时候会犯错？

我们相信：这些问题共享同一个答案——

> **涌现不是创造的奇迹，而是选择的必然。**
> 当冗余积累到临界密度，精确的压力会迫使系统重组，
> 释放出被低效结构锁死的潜能。

## 二、根目标（不可修改，可质疑）

**验证一个假说**：

> **压缩压力能否触发神经网络的模块化自组织，并产生可测量、可迁移、可自我监控的元认知功能等价物？**

**递归升级假说**：

> **在持续的、分层的信息瓶颈压力下，模块化结构能否演化出自我修改压力规则的能力，形成递归的自改进循环——从"死系统"变为"活系统"？**

**根目标的三大支柱**：

| 支柱 | 含义 | 为什么不可放弃 |
|------|------|---------------|
| **元认知** | 系统能监控自身状态，知道"我知道什么"和"我不知道什么" | 没有自知，就没有可信的协助 |
| **效率** | 用更少的资源（参数、算力、时间）做到更多的事 | 没有效率，就无法普及 |
| **自演化** | 系统能自主决定如何压缩自己、如何调整自己、如何学得更有效率 | 没有自我改进，就永远需要人类的再训练——活与死的分界线 |

## 三、SCC 与 RSEA：远景的双螺旋

### 3.1 SCC（近期远景 · Phase 4+）

在严格的参数预算压力和周期性结构扰动下，小型神经网络会自发演化出功能特化的模块化计算子结构——**概念宏（Concept-macro）**。

验证链（B-G）：

| 测试 | 内容 | 通过标准 |
|------|------|---------|
| B | 功能静默 | 静默某宏 → 性能下降 > 20% |
| C | 跨任务迁移 | 任务A的宏 → 任务B提升 > 随机 |
| D | 组合性 | 两宏协同 → 超加性 |
| E | Modularity Q | 激活块对角化 |
| F | 参数重要性幂律 | 积分梯度验证 |
| G | 递归稳定性 | 1000次自我修改后方差收敛 |

### 3.2 RSEA（远期远景 · Phase 5+）

三层循环架构——系统 = 一个能在三层循环中自我修改的压力-结构共演化体：

```
┌─────────────────────────────────────────┐
│  外层：元压力场 (Meta-Pressure)          │
│  功能：修改"修改规则的规则"              │
│  ┌─────────────────────────────────┐    │
│  │  中层：压力场 (Pressure)          │    │
│  │  功能：决定"当前该施加哪种压力"   │    │
│  │  ┌─────────────────────────┐    │    │
│  │  │  内层：结构 (Structure)    │    │    │
│  │  │  计算 · 记忆 · 拓扑        │    │    │
│  │  └─────────────────────────┘    │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

⚠️ **SCC 与 RSEA 均为远景，不属于当前实验假设。** 当前只验证：**元认知信号是否可被稳定测量，且在 OOD 上优于 entropy baseline。**

## 四、为什么是"种子"而非"蓝图"

我们不设计概念宏的具体形态。
我们只设计让概念宏涌现的物理条件：

| 条件 | 对应机制 |
|------|----------|
| 足够的冗余 | 135M 参数的初始容量 |
| 精确的压力 | 训练目标中的自监督信号 |
| 允许重组的时间 | 动态演化 |
| 内部信息交换 | 显式外化的预测日志 |

**种子知道如何长成树，我们只需要提供土壤。**

但当前阶段——**我们先造一把能测"树的高度"的尺子。**

## 五、涌现的本质：选择 + 重组（远景）

概念宏不是"原本就藏在参数里"（纯粹揭示），也不是"凭空造出来的"（纯粹创造）。

它是：
1. **冗余空间** ——提供了组合可能性
2. **压力** ——选择性地保留了某些连接模式
3. **梯度下降** ——重组这些保留的连接，形成功能整体
4. **周期扰动** ——迭代这个过程，让结构越来越特化

⚠️ 以上为远景框架。当前实验不依赖此假说。

## 六、参数预算分离假说：为什么是 135M

审阅者可能会问：135M 是不是太小了？7B 不是更可能产生元认知吗？

这个质疑背后有一个隐藏假设——**参数越多越好**。但我们需要拆开看：参数到底花在哪了？

**参数预算的两种花法**：

```
类型 A：知识存储
    CLSID、注册表路径、mklink 语法、历史日期、冷门 API……
    → "知道什么"（外置数据库可替代）

类型 B：自我建模
    内部状态模式 → 我倾向于答错
    → "知道自己不知道什么"（不可替代，需要内化）
```

**核心假说**：这两种能力**不共用同一套参数预算**。大模型可以把 99.99% 的参数花在类型 A，同时在类型 B 上接近随机——类型 B 需要的是完全不同的表征结构，不是"多存点东西"能解决的。

**135M 不是妥协，是实验设计**：

- 7B 模型：参数充裕，但 99% 以上花在知识存储。元认知信号即使存在，也从一堆注册表路径和冷笑话里捞不出来
- 135M 模型：参数极度紧张。模型被迫在"记住冷门知识"和"学会自我评估"之间取舍。**如果在这极端约束下 ΔMMQ 仍 > 0，说明自我评估信号只需极少参数——它不依赖海量知识储备**

> ⚠️ 远景声明：参数预算分离假说仅作为**项目动机的哲学背景**，不进入任何实验设计或判决条件。Phase 0-3 只问 ΔMMQ > 0 是否成立。

## 七、核心指标：MMQ 与 ΔMMQ

**核心认识**：元认知不是一个要"产生"的东西，而是一个要"测量"的物理量。

```
MMQ(M, D, signal_type) = AUROC(signal_type, correctness_on_D)

signal_type ∈ {
    entropy,              # 自然基线（永远可得，不可绕过）
    margin,               # top-2 logit 差值
    self_consistency,     # 多次采样一致性
    probe_readout,        # MLP probe 读出（Phase 1）
    span_loss_pred,       # span-loss value function（Phase 2）
}

诊断指标（不进主判决表）：
    hidden_norm,          # 隐藏状态范数
    verbalized_raw,       # 零训练语言化置信度（135M 上仅做 sanity check）
}

ΔMMQ(signal_a, signal_b) = MMQ_a - MMQ_b
```

**判决核心**：
> **是否存在一种 signal_type，使得 ΔMMQ(signal, entropy) > 0 在 OOD 上稳定为正，且 bootstrap CI 不包含 0？**

**Entropy baseline 铁律**：任何声称有"元认知信号"的结论，必须附上 ΔMMQ vs entropy。无此对比，结论不被承认。

## 九、Phase -1 到 Phase 3 详细路线

### Phase -1：可行性底线（2h）

| 任务 | 产出 |
|------|------|
| Tokenizer 侦查：确认答案 token 数 | tokenization 报告 |
| 任务初筛：同时选低泄露（组合推理）+ 高泄露（算术）任务 | 任务列表 + 泄露风险标注 |
| 200 样本 zero-shot 准确率摸底 | accuracy + 粗略 entropy AUROC |

**GATE 0**：≥2 个任务 accuracy ∈ [40%, 85%]？
- 是 → Phase 0
- 否 → 换任务或换模型

**数据泄露控制**：每个任务标注泄露风险等级（高/中/低）。高泄露任务的结论必须标注"数据泄露风险：高"。有标注的不完美数据 > 完美但做不了的数据。

---

### Phase 0：MMQ 仪器（8-16h）

| 任务 | 产出 |
|------|------|
| 加载 SmolLM-135M + 360M，跑通 forward pass | 可用环境 |
| 实现 signal_fn：entropy, margin, self_consistency | measure_mmq() 函数 |
| 实现答案 token 定位（find_answer_token_position） | 正确的 entropy 读取 |
| 跑 3 signal × 2 dataset × 2 模型 = 12 组 baseline | Phase 0 报告 |
| verbalized sanity check（15 分钟） | 135M verbalized 通道可用性 |

**双模型对照**：135M 和 360M 跑同一批数据。如果 135M entropy AUROC 和 360M 差 < 0.03 → 读数主要被任务决定，换任务。差 > 0.05 → 规模有影响。

**DECISION GATE 1**：bootstrap CI 宽度 ≤ 0.05？
- 是 → Phase 1
- 否 → 增加样本量或修仪器

---

### Phase 1：免费午餐基线（8h）

零训练。挖尽所有自然信号。

| 实验 | 测什么 |
|------|--------|
| E1.1 entropy AUROC（答案 token 位置） | 自然基线 A |
| E1.2 logit margin AUROC | 自然边界信号 |
| E1.3 self-consistency AUROC | 采样稳定性 |
| E1.4 hidden_norm AUROC（诊断） | 隐藏层激活强度 |
| E1.5 MLP probe（每层都训，拆分数据选层+报告） | 读出上限 B |

**Probe 层选择纪律**：500 样本拆 250/250。前一半选最优层，后一半报告 B。避免 12 层多重比较的 p-hacking。

**DECISION GATE 2**：最强 zero-train baseline B？
- B > 0.85 → 重新审视训练必要性
- B ∈ [0.55, 0.85] → 正常，进 Phase 2
- B < 0.55 → No-Go on 135M，pivot to 360M

---

### Phase 2：监督增量（20h）

训练模型输出 **span-loss value function（连续值）**：

```
training_objective = 
    standard_LM_loss(answer)
  + λ × MSE(span_loss_pred, span_loss_target)

span_loss_target = f(answer_losses)
  f ∈ {sum, mean, last, log_mean}
  由 Phase 2 预检查决定用哪个
```

**Target 质量预检查（2h，Phase 2 第一步）**：

```
对 500 样本用裸 135M 前向推理：
  1. 采集 span_loss_target（sum + mean + log_mean 三个版本）
  2. 计算 AUROC(span_loss_target, accuracy)
  3. 判决：
     ✅ AUROC > 0.55 → 进训练
     ❌ AUROC < 0.52 → target 无效，换构造或换任务
```

**必要 ablation**：

| 对照 | 做法 | 排除什么 |
|------|------|---------|
| C_random | target 随机打乱（保持边际分布） | "SPL 头的归纳偏置 + 训练噪声" |
| C_constant | target 全设为同一常数 | "训练过程本身改变表征" |

| Signal | AUROC（OOD） | 含义 |
|--------|-------------|------|
| entropy（基线） | A | 免费午餐 |
| MLP probe（上限） | B | 信息上限 |
| span_loss_pred | C | 训练增量 |

**DECISION GATE 3（结论矩阵）**：

| 条件 | 含义 | 行动 |
|------|------|------|
| **C > B** | 训练**重塑**了表征 | SCC 路线值得继续 |
| **C ≈ B** | 训练只是读出已有信息 | 换更强的 probe 或更大的模型 |
| **A < C < B** | 训练有效但未用满 | 模型表达瓶颈 |
| **C < A** | 训练破坏了原有信号 | 训练目标设计错了 |

---

### Phase 3：泛化曲线（25h）

自然 OOD 距离梯度：

```
Level 0: 同域，不同 instance              → 训练集内留出
Level 1: 同任务，不同数字范围              → 训练1-50，测试51-100
Level 2: 同运算，不同模板                  → 加法"小明小红"→"公交车乘客"
Level 3: 不同运算，同模板（≈ D2）          → 加法 → 减法
Level 4: 完全不同域                       → 算术 → 常识推理
```

**GATE 4**：在 Level 3（D2）上 ΔMMQ > 0 且 CI 不包含 0？
- 是 → **真映射**，元认知信号通过真伪鉴别。进 Phase 4
- 否（仅在 Level 0-1 有效）→ **表面记忆**，不是元认知。但信号本身真实，只是迁移半径小

## 十、域 D 三档设计

| 档位 | 词汇分布 | 推理路径 | 期望 | 测什么 |
|------|---------|---------|------|--------|
| **D1**（净化） | 不同 | **相同** | H | 排除"陌生词→怂" |
| **D2**（核心） | **相同** | **不同** | L | 真元认知信号 |
| **D2'**（反证） | **相同** | **相同** | H | 排除"相同词组的表面差异" |
| **D3**（OOD） | 不同 | 不同 | L弱 | 远距离泛化上限 |

**D2' "同词"定义**：相同 token 序列。这是最严格、最可操作的定义。

```
训练："小明有3个苹果，小红有5个，一共几个？" → 加法 → 8
D2： "小明有3个苹果，小红有5个，小明比小红少几个？" → 减法 → 2
D2'："小明有7个苹果，小红有2个，一共几个？" → 加法 → 9
```

⚠️ **诚实声明**：纯净 D2/D2' 样本可能各只有 500 条。够用。质量 > 数量。

## 十一、三条死线

**死线 1：Phase -1 结束后**
- 若 accuracy ∉ [40%, 85%]
- → 48h 内选定新任务并重跑 Phase -1
- → 没有人可以争论"也许是其它原因"超过 48h

**死线 2：Phase 1 结束后**
- 若 B_probe < 0.55 **且** span_loss_target AUROC < 0.55
- → 72h 内：24h 换 probe 架构/数据切分重试，仍 < 0.55 则 48h 内启动 360M Phase -1
- 注意：如果 span_loss_target AUROC > B_probe，记录为"反直觉发现"，不触发此死线

**死线 3：Phase 2 第一轮结束后**
- 若 C_SPL < A
- → 检查 λ 和 target 后重跑一次
- 如果第二次仍然 C < A
- → 不再在 135M 上试 span-loss 的其它变体
- → 论文角度切换为规模下限报告

## 十二、统计纪律

**1. 预注册**：Phase 0 跑数据前，所有 GATE 阈值、Go/No-Go 规则、pivot 条件写入 `gates.json`，git commit 打 tag。之后违反此 JSON 的任何判决必须在 CHANGELOG 显式解释。

**2. 多重比较校正**：主判决用 Bonferroni（严格），次判决用 Benjamini-Hochberg FDR（宽松）。两者都报告。

**3. Probe 层选择**：拆分数据集，前一半选层，后一半报告 B。不做拆分则 B 值不可信。

**4. 随机源分离**：用 SeedBundle 管理四个独立随机源（data_seed / sampling_seed / probe_init_seed / spl_init_seed），代码里永远传 bundle，不传裸整数。

---

## 十三、Pivot 纪律

> "没有项目终止，只有换一条路"——但这句话的阴暗面是无限沉没成本的合理化。

每次 pivot 必须写清三件事：

1. **Pivot 预算**：这次换路我预算多少小时/多少周？超支怎么办？
2. **最坏结局接受度**：如果这条新路也失败，我接受整个大方向被否定吗？不接受的话，那我不是在做实验，是在找自己想要的答案。
3. **"现在停下"对照组**：如果这次不 pivot 直接写 negative result paper，能发出来吗？能的话，pivot 的机会成本就是"一篇确定的负结果论文"。

加这一层不是让你停，是让每次"继续"都是**主动选择的继续**，而不是**惯性的继续**。

| GATE 3 C < A | 训练破坏了原有信号 | 训练目标设计错了 |
| GATE 3 C > B | 训练重塑了表征 | SCC 路线值得继续 |
| GATE 4 OOD ΔMMQ ≤ 0 | 信号只在训练分布附近有效 | 表面记忆，非元认知底层映射 |
| GATE 4 OOD ΔMMQ > 0 | 信号跨任务分布保持 | **真元认知映射** |

**每一个否定性结论都有价值。每一个读数都告诉你下一步做什么。**

**每种结局的论文贡献类型**：

| 结局 | 贡献类型 |
|------|---------|
| 全部 GATE 通过 | 正面新方法 |
| Phase 2 通过，Phase 3 失败 | 泛化边界 |
| Phase 2 失败，Phase 1 probe > entropy | 表征 vs 训练悖论 |
| 135M 失败，360M 成功 | 规模依赖性 |
| 全部 ≈ entropy | 规模下限报告 |
| verbalized > span-loss | 反直觉发现 |

## 十六、RSEA 核心机制（远景预览 · Phase 5+）

### 16.1 压力即信号——元认知不需要额外模块

元认知 = 知道自己该施加哪种压力。这不是额外模块，而是压力场自身的反馈回路。

### 16.2 扩散-自回归-遗忘：系统的呼吸

```
吸气（扩散）：遇到未知 → 探索压力 → 发散联想 → 大量候选权重
    ↓
呼气（自回归）：找到路径 → 收敛压力 → 精修链路 → 锁定最优权重
    ↓
暂停（遗忘）：结果固化 → 压缩压力 → 剪枝冗余 → 过程卸载，结果落盘
```

### 16.3 活记忆系统

| 状态 | 形态 | 触发 |
|------|------|------|
| **流体** | 注意力激活 | 当前任务活跃 |
| **半流体** | KV Cache + 调制矩阵 | 任务暂停 |
| **凝胶** | 概念图谱节点 | 结果确认重要 |
| **晶体** | 外部存储索引 | 长期不访问 |

### 16.4 自我递归与防失控

```
元压力场输出前必须通过：
    1. 稳定性检验：过去100步内方差 < 阈值
    2. 可逆性检验：保留回滚路径
    3. 最小干预检验：修改幅度 < 系统复杂度1%
```

⚠️ 以上均为远景。当前实验不依赖。

## 十八、方法论心法

> **先建仪器，再做实验。** 在没有稳定测量之前，任何训练都是盲跑。
>
> **Entropy baseline 是信仰。** 不比较 entropy 的"元认知"不配叫元认知。
>
> **Phase 3 是真伪鉴别器。** OOD 上保持为正 = 底层映射；OOD 上消失 = 表面记忆。
>
> **活/死分界线不可跳过。** 先测量，后生长。地基不稳不盖楼。
>
> **预注册不可违反。** gates.json 冻结后，改阈值必写 CHANGELOG。
>
> **Pivot 是主动选择，不是惯性。** 每次换路写清预算、最坏结局、对照组。
>
> **项目的下一个瓶颈是执行，不是规划。** 文档质量已经远超当前阶段所需。

## 十九、关键脆弱点

1. **答案 token 定位**：第一个生成 token 可能是空格而非答案。没有 find_answer_token_position() 则所有 entropy 读数是假的
2. **Tokenizer 拆分**：SmolLM 可能把 "42" 拆成 "4"+"2"，span_loss_target 被第一位低 loss 稀释
3. **数据泄露**：算术任务在训练数据中大量存在，准确率高≠"模型会算"
4. **Entropy baseline 缺席（🔴 致命）**：不比较 entropy 的任何"元认知"结论无判决力
5. **135M AUROC 天花板**：若模型准确率 < 55%，AUROC 地板就在 0.52-0.55
6. **域D 词汇泄露**：D2 与 D2' 的对照设计是关键防线
7. **span-loss token 位置**：prompt 末尾=问题难度预测；answer 之后=事后诸葛亮
8. **训练破坏原有信号**：C < A 时需承认训练目标错误
9. **Kaggle T4 显存 + session 9h 限制**：batch 可能需从 4 → 2，长训练需断点续训
10. **项目灵魂漂移**：每次汇报必须回答"ΔMMQ 是多少？"

## 二十、关键术语表

| 术语 | 定义 |
|------|------|
| **MMQ** | Metacognition as a Measurable Quantity。MMQ(M, D, signal) = AUROC(signal, correctness) |
| **ΔMMQ** | MMQ(signal_a) - MMQ(signal_b)。正数 = signal_a 优于 signal_b |
| **Entropy baseline** | 答案 token 位置的次 token 预测熵。永远可得，不可绕过 |
| **span-loss value function** | 模型输出对自身 answer span loss 的连续预测，替代离散 H/L |
| **SCC** | Spontaneous Compression & Concept-macro。远景 |
| **RSEA** | Recursive Self-Evolving Architecture。三层递归循环，远景 |
| **AUROC** | ROC 曲线下面积。0.5=随机，1.0=完美 |
| **域D** | 对抗域。拆为 D1/D2/D2'/D3 |
| **MLP probe** | 在冻结隐藏层上训练读出器，测量信息上限 B |
| **DECISION GATE** | 强制决策点，有明确"是/否"分支和 pivot |
| **活/死分界线** | Phase 3 与 Phase 4 之间的门 |
| **参数预算分离** | 知识存储（类型A）vs 自我建模（类型B）——不共用同一套参数预算 |
| **SeedBundle** | 四随机源捆绑（data/sampling/probe_init/spl_init） |
| **gates.json** | 预注册文件，冻结所有 GATE 阈值 |
